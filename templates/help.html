Translation states

Translation items can be in 4 states:

  • translated: everything is fine
  • needs update: the translation should be updated because the meaning of the msgid evolved
  • needs review: the translation might be incorrect and should be reviewed
  • needs translation: the item is untranslated

    Translation comments

    For each translation, you have a general comment (generally to explain what the msgid reflects), and one comment per language, for comments specific to this language

    Which translation is being actually used?

    If a string has a translation (aka not 'needs translation'), this version is used. Else, it tries to get the best available translation for other languages. For example, if the catalan version is not available, it will try to use the spanish one, then the english one, and so on

    Where can I find the msgid for a string?

    Have a look at the i18n directory from the subversion repository

    How can I add new items?

    Only moderators can add new translation items

    Typography

    Only french uses a space before ':' or '!'. A non breakable space can be inserted with '&nbsp;', but be aware that you shouldn't use it for each string.
