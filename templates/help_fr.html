États de traduction

Les chaînes de traduction peuvent être dans 4 états différents :
  • translated (traduit) : la traduction est valide
  • needs update (à mettre à jour) : la traduction doit être mise à jour car le sens de la chaîne a changé
  • needs review (à vérifier) : la traduction est peut être incorrecte et nécessite une vérification
  • needs translation (à traduire) : la chaîne n'est pas traduite

Le texte entré dans un champ avec un état 'needs translation' (à traduire) n'a pas d'effet.

Une fois que vous avez traduit / vérifié / mis à jour une chaîne, n'oubliez pas de mettre à jour sont état.

Commentaires de traduction

Pour chaque chaîne, vous disposez d'un commentaire général (utilisé pour expliquer le sens de la chaîne à traduire), et d'un commentaire par langue pour des besoins spécifique à chaque langue.

Quelle traduction est utilisée ?

Si une chaîne a une traduction (c'est à dire qu'elle a un statut différent de 'needs translation'), celle-ci est utilisée. Sinon, la meilleure traduction dans les autres langues est utilisée. Par exemple, si la version catalane n'existe pas, la version espagnole sera utilisée, ou à défaut la version anglaise, sinon française, etc.

Où puis-je trouver le msgid de la chaîne correspondant à une traduction ?

Aidez-vous du répertoire i18n dans le dépôt subversion.

Comment puis-je ajouter une nouvelle chaîne ?

Seuls les développeurs peuvent ajouter une nouvelle chaîne.

Typographie

Seul le français utilise un espace devant ':' ou '!'. Un espace insécable peut être inséré avec '&nbsp;', mais utilisez le avec précaution, il n'est pas nécessaire pour toutes les chaînes.

Notez que certaines chaînes peuvent contenir un espace en dernière position (ce cas est rare). Pensez à ajouter un commentaire général sur la chaîne afin que les traducteurs ne l'oublient pas.
